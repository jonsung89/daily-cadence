# Daily Cadence: Design & Branding Guide

## Brand Identity

### Name
**Daily Cadence**

### Tagline
"Log your life, find your beat"

### Brand Philosophy
Daily Cadence is modern, minimal, and human. It feels premium but approachable. The design should feel like a well-crafted journal, not a corporate app. Inspired by Anthropic's clean, earthy aesthetic—sophisticated without being cold.

---

## Visual Direction

### Overall Feel
- **Modern minimalism** – Clean lines, generous whitespace, purposeful elements
- **Warm & inviting** – Earthy tones, human touch, not sterile
- **Intuitive** – Visual hierarchy is clear; users understand what to do without explanation
- **Customizable** – The default looks good, but users should feel ownership
- **Progress-focused** – Charts, progress bars, and visualizations are celebrated, not hidden

### Inspiration
Anthropic Academy aesthetic: neutral, earthy, professional yet approachable. Think: calm, focused, smart.

---

## Color Palette

### Primary Colors (Default Theme)

**Warm Beige/Cream** (Primary Background)
- Hex: `#F5F3F0` (or similar warm off-white)
- Used for: Main background, card backgrounds
- Feeling: Clean, warm, inviting

**Deep Sage/Olive** (Primary Action & Accents)
- Hex: `#5A7B6D` (or similar muted green-gray)
- Used for: Buttons, links, primary actions, accent text
- Feeling: Grounded, calm, intentional

**Warm Gray** (Secondary Text & Borders)
- Hex: `#8B8680` (or similar warm gray)
- Used for: Secondary text, dividers, subtle borders
- Feeling: Balanced, not cold

**Near-Black** (Primary Text)
- Hex: `#2C2620` (or similar dark brown-gray)
- Used for: Body text, headlines
- Feeling: Easy to read, warm tone

**Light Taupe** (Hover States & Subtle Backgrounds)
- Hex: `#EAE6E1` (or similar light warm gray)
- Used for: Hover states, subtle background fills, inactive states
- Feeling: Gentle, not jarring

### Semantic Colors (for Note Types)

These are the default color suggestions for note types. Users can customize these.

**Workout**
- Hex: `#C85A54` (warm terracotta red)
- Alternative: `#D97C6E`

**Meal**
- Hex: `#E8B86B` (warm gold)
- Alternative: `#F0C680`

**Sleep**
- Hex: `#4A6B8A` (muted deep blue)
- Alternative: `#5A7BA0`

**Mood/Feelings**
- Hex: `#A584D3` (soft purple)
- Alternative: `#B59FE0`

**Activity/Steps**
- Hex: `#6BA67B` (muted green)
- Alternative: `#7FB590`

**General/Notes**
- Hex: `#8B8680` (warm gray)

### Color Usage Rules
- Use primary color (sage) for all actionable elements (buttons, links, important CTAs)
- Use semantic colors to distinguish note types in timeline, calendar, and tags
- Maintain at least 4.5:1 contrast ratio for accessibility
- Avoid pure black (#000) and pure white (#FFF)—use warm versions instead
- Dark mode: Invert the palette but maintain warmth (beige background becomes dark charcoal with warm undertones)

## Typography

### Font Families

**Headlines & Titles** (Brand Voice)
- Font: `Playfair Display` (serif, elegant, readable at large sizes)
- Fallback: Georgia, serif
- Weight: 600–700 (semi-bold to bold)
- Use for: App name, section headers, prominent callouts
- Feeling: Sophisticated, intentional, memorable

**Body & UI Text** (Clarity & Legibility)
- Font: `Inter` (sans-serif, neutral, highly legible)
- Fallback: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif
- Weights: 400 (regular), 500 (medium), 600 (semi-bold)
- Use for: Body text, buttons, labels, small copy
- Feeling: Modern, clean, accessible

### Type Scale

**Desktop & Web**
```

H2 (Section Title)  Playfair Display, 36px, 700, line-height 1.2
H3 (Subsection)     Playfair Display, 28px, 600, line-height 1.3
H4 (Card Title)     Inter, 20px, 600, line-height 1.4
Body Large          Inter, 18px, 400, line-height 1.6
Body Regular        Inter, 16px, 400, line-height 1.6
Body Small          Inter, 14px, 400, line-height 1.5

```

**Mobile App**
```

H2 (Section Title)  Playfair Display, 24px, 700, line-height 1.2
H3 (Subsection)     Playfair Display, 20px, 600, line-height 1.3
H4 (Card Title)     Inter, 18px, 600, line-height 1.4
```

### Typography Rules
- Headlines should be warm and inviting, not corporate
- Body text should prioritize readability (generous line-height, adequate contrast)
- Keep color consistent with brand palette (avoid pure black)
- Limit to 2–3 typeface sizes on any screen to maintain clarity

## Logo & Icon System

### Logo Concept

**Primary Logo (Horizontal)**
- Name: "Daily Cadence" in Playfair Display, 700
- Icon to left of text: A simple, rhythmic symbol
- Icon concepts (choose one):
  1. **Heartbeat/Pulse line** – represents the daily rhythm, health focus
  2. **Timeline dot-to-dot** – represents the day's journey, progression
  3. **Music note with upward arrow** – represents cadence + progress
  4. **Sine wave or curved line** – represents rhythm, smooth flow

**Recommended:** Heartbeat or sine wave icon (minimal, memorable, ties to the cadence concept)

- Icon color: Sage green (#5A7B6D)
- Text color: Near-black (#2C2620)
- Usage: Website header, app launcher, social media

### Logo Lockup Variations
1. **Horizontal** (icon + text side-by-side) – for headers, wide spaces
2. **Vertical** (icon above text) – for mobile app icon, narrow spaces
3. **Icon only** (just the symbol) – for app icon, favicon, small spaces

### App Icon (Mobile)

**Shape:** Rounded square (like modern iOS/Android apps)

**Design:**
- Background: Warm sage gradient (light sage to medium sage)
- Icon element: Simple heartbeat line or timeline in white/cream
- Ensure clear at 1x, 2x, 3x resolutions and as small as 180x180px

**File formats:** PNG (iOS), PNG/WebP (Android)

### Icon System

**Navigation Icons**
- Timeline (home) – vertical line with dots
- Progress/Stats – bar chart or trend line
- Settings – gear
- Profile/User – person icon
- Back – left arrow

**Note Type Icons** (user-customizable, defaults provided)
- Workout – dumbbell
- Meal – fork & knife
- Sleep – moon or pillow
- Mood – smiley face or heart
- Activity – person running
- General – pencil or note

**Action Icons**
- Add/Create – plus (+)
- Edit – pencil
- Delete – trash
- Search – magnifying glass
- Filter – funnel
- Calendar – calendar grid
- Camera – camera
- Photo – image icon

**Style:** Minimal line icons, 2–3px stroke weight, no fills. Icons should feel friendly and modern, not sterile.

**Icon Grid:** 24x24px base (scale to 16px, 32px, 48px as needed)

## UI Components & Patterns

### Buttons

**Primary Button**
- Background: Sage green (#5A7B6D)
- Text: Cream/off-white (#F5F3F0)
- Text weight: Inter, 500
- Padding: 12px 24px (mobile), 14px 28px (desktop)
- Border radius: 8px
- Hover state: Slightly darker sage (darken by 10%)
- Disabled state: Light taupe with lighter text
- Focus state: Outline in sage, 2px, offset 2px

**Secondary Button**
- Background: Light taupe (#EAE6E1)
- Text: Sage green (#5A7B6D)
- Text weight: Inter, 500
- Padding: Same as primary
- Border radius: 8px
- Hover state: Slightly darker light taupe
- Disabled state: Even lighter taupe with lighter text

**Text Button** (Link)
- Text: Sage green (#5A7B6D)
- Text weight: Inter, 500
- Underline: Optional, appears on hover
- No background

### Cards

- Background: White or light cream
- Border: Subtle 1px border in light taupe (#EAE6E1) or no border (shadow instead)
- Shadow: Soft drop shadow (0 2px 8px rgba(0,0,0,0.08))
- Padding: 16px (mobile), 20px (desktop)
- Border radius: 12px
- Hover state (if clickable): Slight lift (shadow deepens), background slightly darker

### Timeline Entry (Mobile App)

- Time indicator (left): Sage green dot, 12px diameter
- Connecting line: Subtle taupe line connecting dots
- Content area: Card-style with note type color as accent strip (left edge, 4px wide)
- Note type tag: Small pill with semantic color background, white text
- Text content: Body text in near-black
- Edit/delete actions: Appear on swipe or long-press (iOS/Android patterns)

### Progress Chart

- Chart background: Cream/white
- Chart line: Sage green, 2–3px stroke
- Data point: Small sage circle, 6px diameter
- Hover tooltip: Light taupe background, near-black text
- X-axis/Y-axis: Subtle warm gray lines, light weight
- Grid lines (optional): Very light taupe, minimal
- Legend: Inter, 12–14px, warm gray text

### Dashboard Widget

- Title: Inter, 16–18px, 600 weight, near-black
- Value: Playfair Display or large Inter, bold
- Icon: Semantic color icon (matches the widget type)
- Background: Card style
- Tap/clickable: Slight lift on hover

### Input Fields

- Border: 1px warm gray (#8B8680) or light taupe
- Background: White or very light cream
- Focus state: Sage green border, 2px
- Placeholder text: Warm gray, lighter weight
- Label: Inter, 14px, 500 weight, near-black
- Padding: 12px (horizontal), 10px (vertical)
- Border radius: 8px

- Secondary text: Medium taupe (`#B5ADA5`)

- Borders: Subtle warm gray (`#4A4440`)

### Dark Mode Implementation
- Use system preference to detect (prefers-color-scheme)
- Provide toggle for user override
- Maintain contrast ratios (4.5:1 minimum for text)
- Don't invert colors directly—adjust tones to maintain warmth and readability

### Mobile App Screens

**1. Timeline (Home)**
- Top safe area: Status bar
- Header: "Today" + date
- Optional: Dashboard stats (customizable widgets)
- Main: Vertical timeline of notes
  - Each entry shows: time, note type color bar, type tag, text preview, optional photo
  - Bottom spacing for floating action button
- Bottom: Floating action button (+ icon) to create new note

**2. Create/Edit Note**
- Header: Back button + "New Note" or "Edit"
- Form: 
  - Note type selector (horizontal scroll or dropdown)
  - Text input (full-screen textarea)
  - Optional: Photo picker button
  - Optional: Date/time picker (collapsed, expandable)
  - Optional: Quick-add fields (workout: add exercise, meal: add macros, etc.)
- Bottom: Save button (full width, primary color)

**3. Note Detail**
- Header: Back button + edit/delete actions
- Content: Full note text
- Media: Photos/videos in gallery view
- Type-specific details (e.g., exercises for workout type)
- Edit button

**4. Progress Chart (Exercise/Metric Specific)**
- Header: Exercise name (e.g., "Squats")
- Chart area: Line chart showing progression over time
- Legend/stats below: Max weight, avg reps, date range
- Table: Sortable list of all logged instances
- Back button to previous view

**5. Calendar View**
- Header: Month/year selector (left/right arrows)
- Calendar grid: Days with indicators (colored dots/icons for note types)
- Tap day: See notes from that day
- Legend: What each color/icon represents

**6. Stats Dashboard**
- Header: "Your Progress"
- Customizable widgets (user arranges via drag/drop or settings)
- Widget types: Single stat, progress chart, calendar heatmap, list view
- Edit button: Opens customization mode

**7. Settings**
- Sections: 
  - Note Types (customize colors, icons, names)
  - Dashboard (choose which stats to show)
  - General (notifications, privacy)
  - About (version, links)
- Color picker for custom note types (palette selector + custom hex input)
- Icon selector for custom note types
- Toggle for dark mode

### Web (Dashboard)

**1. Home/Dashboard**
- Left sidebar: Navigation (Home, Workouts, Nutrition, Sleep, Mood, Settings)
- Main content area: Scrollable dashboard with user-selected widgets
- Widgets larger on desktop (2–3 columns)
- Top bar: Search, user profile, settings

**2. Timeline View**
- Left: Sidebar with filters (date range, note type filters)
- Main: Full-width timeline (horizontal, with more detail than mobile)
- Right panel (optional): Edit/details panel (shows full note when selected)

**3. Progress Analytics**
- Left: Sidebar with metric selector (Exercises, Meals, Sleep, etc.)
- Main: Large charts, graphs, tables
- Filters: Date range, specific exercise/meal/category

**4. Exercises/Library**
- List or grid view of all exercises
- Search and filter by muscle group

- Edit/delete options

## Accessibility Guidelines

### Color Contrast
- All text must meet WCAG AA standard (4.5:1 for body text, 3:1 for large text)
- Don't rely on color alone to convey information (use labels, icons, text)
- Semantic colors should be distinguishable for colorblind users (avoid red-green-only combinations)

### Typography
- Minimum 16px for body text on mobile
- Line-height at least 1.5 for body text
- Avoid justified text alignment
- Use adequate letter-spacing (avoid cramped text)

### Interactive Elements
- Minimum touch target size: 48x48px on mobile
- Clear focus states (visible outline, not just color change)
- Logical tab order
- Descriptive button labels (avoid "Click here")

### Motion
- Respect prefers-reduced-motion setting
- Limit animations to non-essential transitions
- No auto-playing videos or flashing content

### Semantic HTML / ARIA
- Use proper heading hierarchy (h1, h2, h3)
- Use semantic tags (button, input, label, form)
- Add aria-labels where needed (icon buttons, form fields)
- Ensure form labels are associated with inputs
